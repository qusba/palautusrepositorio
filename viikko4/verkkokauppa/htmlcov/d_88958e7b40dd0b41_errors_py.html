49class Unsatisfiable(HypothesisException): 

50 """We ran out of time or examples before we could find enough examples 

51 which satisfy the assumptions of this hypothesis. 

52 

53 This could be because the function is too slow. If so, try upping 

54 the timeout. It could also be because the function is using assume 

55 in a way that is too hard to satisfy. If so, try writing a custom 

56 strategy or using a better starting point (e.g if you are requiring 

57 a list has unique values you could instead filter out all duplicate 

58 values from the list) 

59 """

62class Flaky(HypothesisException): 

63 """This function appears to fail non-deterministically: We have seen it 

64 fail when passed this example at least once, but a subsequent invocation 

65 did not fail. 

66 

67 Common causes for this problem are: 

68 1. The function depends on external state. e.g. it uses an external 

69 random number generator. Try to make a version that passes all the 

70 relevant state in from Hypothesis. 

71 2. The function is suffering from too much recursion and its failure 

72 depends sensitively on where it's been called from. 

73 3. The function is timing sensitive and can fail or pass depending on 

74 how long it takes. Try breaking it up into smaller functions which 

75 don't do that and testing those instead. 

76 """
